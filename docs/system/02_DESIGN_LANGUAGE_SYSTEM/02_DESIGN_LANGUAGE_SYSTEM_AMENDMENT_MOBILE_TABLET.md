## Version

1.0

## Status

Approved

------------------------------------------------------------------------

# Purpose

This amendment updates the Responsive Design section of the Design
Language System.

Desktop, Tablet and Mobile are **three independent experiences**.

The Mobile and Tablet interfaces must feel like premium applications
running inside the browser rather than compressed desktop websites.

------------------------------------------------------------------------

# Multi-Experience Philosophy

Triads Future is not a responsive website.

It is one product with three unique experiences.

  Device    Experience
  --------- ----------------------------
  Desktop   Premium Editorial Website
  Tablet    Premium Tablet Application
  Mobile    Premium Mobile Application

Each device must be designed independently.

------------------------------------------------------------------------

# Mobile Experience

## Philosophy

The mobile experience should feel like a native application inside the
browser.

The user should forget they are browsing a website.

Borrow interaction quality from premium mobile apps such as Instagram.

**Do not copy Instagram's branding, colors, layouts, or UI.**

Use it only as inspiration for navigation, interaction quality, and
touch-first design.

------------------------------------------------------------------------

## Mobile Navigation

Use a fixed bottom navigation bar.

Never use a side drawer as the primary navigation.

Never compress the desktop navigation.

The navigation must always remain within comfortable thumb reach.

------------------------------------------------------------------------

## Navigation Items

Use meaningful icons together with small labels.

Never use icons alone.

Recommended navigation:

-   Home
-   Solutions
-   Work
-   Insights
-   Book

The **Book** tab routes directly to the Strategy Call section.

------------------------------------------------------------------------

## Icons

Recommended icon family:

-   Lucide Icons

Alternative:

-   Phosphor Icons
-   Tabler Icons

Recommended icons:

  Page        Icon
  ----------- -----------------------
  Home        House
  Solutions   LayoutGrid
  Work        BriefcaseBusiness
  Insights    BookOpen or Newspaper
  Book        Calendar or Phone

Icons must be outlined, monochrome and consistent.

------------------------------------------------------------------------

## Labels

Every icon must include a small label.

Label rules:

-   10--11px
-   Medium weight
-   Sentence case
-   Easy English
-   Secondary text color

Icons remain the primary visual element.

Labels improve clarity.

------------------------------------------------------------------------

## Bottom Navigation Style

-   Fixed to bottom
-   Respect device safe area
-   Rounded top corners
-   Thin top border
-   No glassmorphism
-   No heavy shadows
-   No floating action button

------------------------------------------------------------------------

## Active State

Allowed:

-   Slightly darker icon
-   Slightly darker label
-   Thin indicator

Never use:

-   Bright colors
-   Bounce
-   Glow
-   Oversized pills

------------------------------------------------------------------------

# Tablet Experience

## Philosophy

Tablet is its own experience.

It is not Desktop.

It is not Mobile.

Borrow interaction quality from premium tablet applications.

Do not copy layouts or branding.

------------------------------------------------------------------------

## Tablet Navigation

Use a left Navigation Rail.

Never use a top navigation as the primary navigation.

Navigation rail structure:

Top

-   Logo

Middle

-   Home
-   Solutions
-   Work
-   Insights
-   Book

Bottom

-   Optional CMS/Profile actions

------------------------------------------------------------------------

## Navigation Rail Rules

Every item includes:

-   Icon
-   Small label

The rail should be:

-   Narrow
-   Minimal
-   Touch-first
-   Quiet
-   Always visible

------------------------------------------------------------------------

# Responsive Rules

Desktop

-   Top Navigation Bar

Tablet

-   Left Navigation Rail

Mobile

-   Bottom Navigation Bar

Navigation systems must never be shared across devices.

Each device has its own UX.

------------------------------------------------------------------------

# Final Rule

The inspiration comes from premium mobile applications because they
provide refined interaction patterns.

The final design must remain uniquely Triads Future.

Visitors should think:

> "This feels as polished as a premium app."

They should never think:

> "This looks like Instagram."

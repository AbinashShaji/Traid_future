# 02_DESIGN_LANGUAGE_SYSTEM

# PART 04 — Device Experience System (Desktop • Tablet • Mobile)

**Version:** 1.0

---

# Purpose

Triads Future is **not** a responsive website.

It is one product that delivers three intentionally different experiences.

Each experience is designed around how people naturally use that device.

Never compress Desktop into Mobile.

Never enlarge Mobile into Tablet.

Design separately.

Build separately.

Optimize separately.

---

# Multi-Experience Philosophy

Every device has different user behaviour.

Desktop users explore.

Tablet users browse.

Mobile users act.

The interface should respect these behaviours.

# Desktop Experience

## Philosophy

Desktop is the flagship experience.

It communicates professionalism, confidence and expertise.

The visitor has more space.

The visitor spends more time reading.

The visitor compares options.

The design should encourage exploration.

Editorial composition.

Large margins.

Calm movement.

Social.

# Tablet Experience

## Philosophy

Tablet is not a large phone.

Tablet is not a small desktop.

Tablet is its own product.

The experience should feel like a premium tablet application.

---

Apple Photos

# Mobile Layout

Single column.

---

# Mobile Cards

Cards should feel like application screens.

Comfortable touch.

Large spacing.

Minimal decoration.

---

# Mobile Work Section

Should feel closer to browsing an app than browsing a portfolio website.

Large images.

Comfortable spacing.

Natural scrolling.

---

# Mobile Forms

Minimal required fields.

---

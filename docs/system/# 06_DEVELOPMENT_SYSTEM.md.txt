**Version:** 1.0

# Purpose

This document defines how the Triads Future website must be developed.

It ensures every developer and AI tool writes clean, maintainable, scalable and beginner-friendly code.

This document is the engineering standard for the project.

---

# Development Philosophy

The project should be easy to

- Understand
- Maintain
- Expand
- Debug
- Hand over

Future developers should understand the project within one day.

Never write code only you can understand.

---

# Core Principles

## Principle 01

Readable code is more valuable than clever code.

---

## Principle 02

One file should have one responsibility.

---

## Principle 03

Every important function should be commented.

---

## Principle 04

Never duplicate logic.

Reuse components.

Reuse utilities.

Reuse hooks.

---

## Principle 05

If something is editable,

it belongs inside the CMS.

Never hardcode editable content.

2. Next

3. External Libraries

4. Internal Components

---

# Coding Style

Avoid deeply nested code.

Prefer early returns.

Small functions.

Clear names.

One purpose.

Never use meaningless names.

Bad

data

temp

item

Good

featuredProjects

navigationItems

contactInformation

---

# Comments

Every important function includes

Purpose

Parameters

Returns

Example

```ts
/**
 * Creates a new strategy call request.
 *
 * @param data Strategy call form data
 * @returns Created record
 */
```

Never comment obvious code.

Comment business logic.

---

# TypeScript Rules

Never use

any

Always define interfaces.

Prefer

type

for unions.

interface

for objects.

Strict mode enabled.

---

Priority

# API Architecture

Every feature owns its own API.

Example

/api/work

/api/solutions

/api/contact

/api/insights

/api/cms

/api/admin

Never place everything inside one file.

---

# Database Rules

Prisma is the only ORM.

Never access SQL directly.

Relationships must be normalized.

IDs use UUID.

Soft delete preferred.

CreatedAt

UpdatedAt

Every table includes timestamps.
